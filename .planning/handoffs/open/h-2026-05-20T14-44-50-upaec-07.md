# Phase 07: Online Store MVP — Build Brief

## Goal
Scaffold the UPAEC e-commerce purchase path so customers can order a produce box with add-ons, choose delivery or pickup, select a subscription or one-time purchase, and complete a mobile-first checkout.

## Stack (from PROJECT.md)
- Next.js 15 + React 19 + Tailwind CSS + shadcn/ui (App Router)
- Payload CMS (products catalog + orders admin)
- Commerce: Shopify Storefront API OR custom cart (evaluate and decide during this phase — start with custom if Shopify adds friction)
- Email: Resend or SendGrid (order confirmation)
- Hosting: Vercel

## Data Models (from PROJECT.md — implement these exactly)

```ts
// Payload CMS collection: Products
Product {
  name: string
  description: string
  price: number
  subscriptionEligible: boolean
  oneTimeEligible: boolean
  boxSizeOptions: ('starter' | 'family' | 'custom')[]
  addOnProducts: Product[]  // relation to self
}

// Payload CMS collection: Orders
Order {
  customer: { name, email, phone, zip }
  items: { product: Product, quantity: number, boxSize?: string }[]
  deliveryMethod: 'delivery' | 'pickup'
  deliveryDate: Date
  pickupLocation?: string
  subscriptionType?: 'one-time' | 'weekly' | 'biweekly' | 'monthly'
  status: 'pending' | 'confirmed' | 'fulfilled' | 'cancelled'
  totalCents: number
}
```

## Tasks (implement in order)

### 07-01: Payload CMS Product collection
- `payload.config.ts` — add Products collection
- `collections/Products.ts` — full schema above
- Seed data: Starter box ($25, 6 items), Family box ($35, 9 items), and the ~12 add-on SKUs from `.planning/research/copy/upaec-web-page-info.md` (kale, collards, mustard, chard, southern blend, tomatoes, peppers, onions, beets, potatoes, okra, etc.)

### 07-02: Box-size selector + add-on grid
- `app/(store)/boxes/page.tsx` — renders BoxSizeCard for Starter / Family / Custom
- `components/store/BoxSizeCard.tsx` — price, item count, CTA "Select this box"
- `components/store/AddOnGrid.tsx` — product grid with per-item qty input
- Reads from Payload CMS Products collection via local API

### 07-03: Checkout page
- `app/(store)/checkout/page.tsx`
- `components/store/DeliveryPickupToggle.tsx` — 'Delivery' | 'Pickup' toggle; if Pickup show slot selector placeholder
- `components/store/SubscriptionSelector.tsx` — One-time | Weekly | Bi-weekly | Monthly radio
- Customer info form: name, email, phone, ZIP, optional notes
- Cart summary: selected box + add-ons + subtotal

### 07-04: Mobile-first audit
- Verify 360px / 768px / 1280px clean on boxes page, checkout, and order confirmation
- No overflow, no truncation, touch targets ≥ 44px

### 07-05: Order submission + confirmation
- `app/api/orders/route.ts` — POST handler: validate payload, persist Order to Payload CMS, send confirmation email via Resend
- `lib/email/send-confirmation.ts` — Resend email template with order summary
- `app/(store)/order-confirmation/[id]/page.tsx` — shows order ID, items, total, expected window, contact info

## Design constraints
- Palette: UPAEC cream/charcoal base, sage/tan accents, golden-hour tone (defined in Tailwind theme from Phase 01)
- shadcn/ui primitives only — no Radix direct, no custom component library installs
- Every visible region MUST have a deterministic `cid` + search token via the Visual Context System (packages/visual-context)
- Mobile-first: design at 360px, scale up
